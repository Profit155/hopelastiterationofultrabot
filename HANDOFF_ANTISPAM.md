## Проблема (со слов пользователя)

Бот спамит ВСЕ действия с самого начала тренировки (рандомная
инициализация политики), и этим сам себя гасит:

1. **Тряска прицела.** yaw/pitch постоянно меняют знак — камера
   стоит на месте, но автонаводка ULTRAKILL по врагам сбивается из-за
   дёрганья каждый кадр.
2. **Спам даша.** Даш стопит игрока + жрёт `boostCharge`. Банк
   стамины всегда пуст, контроль ресурса (механика игры) сломан.
3. **Спам кулака.** Аналогично: банк зарядов кулака всегда пуст, +
   бот сам себя стопит анимацией каждую секунду.
4. **Прыжок + слайд → Slamdown.** В воздухе slide = слэм. Бот спамит
   и jump, и slide — каждый второй кадр себя впечатывает в землю.
5. **Whiplash спам.** Гарпун ситуативный, его нельзя жать просто так.
6. **Спам смены оружия.** Перезарядка/стрельба сбиваются.

В текущей `TcpBridge.CalculateReward` ничего из этого не штрафовалось.

## Решение (что сделал)

Идея: штрафовать **факт траты ресурса**, **отказы игры**, **дрожь** и
**антипаттерн комбо**, а не «нажатие как таковое». Все коэффициенты
маленькие, согласованы со шкалой текущей награды (шаг -0.001, килл +1,
парри +3, смерть -10), чтобы осмысленные действия в бою всё ещё
окупались.

### Изменённые файлы

1. **`UltrabotMod/Plugin/ActionExecutor.cs`** — добавлены публичные
   счётчики, обнуляются в начале каждого `Execute()`:

   ```csharp
   public float CameraJitter;     // |Δyaw|+|Δpitch| при смене знака raw input
   public float MoveJitter;       // то же для осей движения
   public int   WastedActions;    // нажал по фронту, но игра отказала
   public int   DashUsed;         // реально вызвали Dodge
   public int   PunchUsed;        // реально вызвали PunchStart
   public int   SlamTriggered;    // реально вызвали Slamdown ИЛИ jump+slide одновременно
   public int   WeaponSwitches;   // реально сменили слот / вариацию
   public int   WhiplashFired;    // реально кинули хук
   public int   FireToggles;      // фронты fire1/fire2 за шаг
   public int   ButtonsHeldCount; // сколько дискретных кнопок (4..20) удерживается
   ```

   - Дрожь камеры: в блоке камеры, при смене знака `rawYaw*_prevRawYaw < 0`
     прибавляется `|rawYaw - _prevRawYaw|`. Аналогично для pitch и
     осей движения.
   - В каждом `if` дискретного действия, который реально что-то
     запускает (jump/dash/slide/slam/whiplash/punch/смена слота/swapVar),
     теперь есть `else` ветка, инкрементирующая `WastedActions`. При
     успехе инкрементируется `*Used / *Triggered / WeaponSwitches`.
   - Отдельно ловится антипаттерн `jump && slide` одновременно по
     фронту → `SlamTriggered++`.
   - На cooldown-блоке свича оружия: пресс по фронту тоже считается
     `WastedActions`.
   - В `ReleaseAll()` обнуляются `_prevRawYaw/Pitch/MoveFwd/Right`.

2. **`UltrabotMod/Plugin/TcpBridge.cs`** — `CalculateReward()`:

   ```csharp
   var ae = _actionExecutor;
   float rewJitter     = -0.03f * ae.CameraJitter - 0.01f * ae.MoveJitter;
   float rewWasted     = -0.02f * ae.WastedActions;
   float rewBank       = -0.05f * ae.DashUsed - 0.05f * ae.PunchUsed;
   float rewSlamAbuse  = -0.10f * ae.SlamTriggered;
   float rewSwitchSpam = -0.05f * ae.WeaponSwitches;
   float rewWhipSpam   = -0.03f * ae.WhiplashFired;
   float rewFireTog    = -0.02f * Mathf.Max(0, ae.FireToggles - 1);
   float rewMash       = -0.01f * Mathf.Max(0, ae.ButtonsHeldCount - 3);
   ```

   Все восемь компонентов прибавлены к общей сумме `reward` и
   проброшены в HUD.

3. **`UltrabotMod/Plugin/DebugHUD.cs`** — добавлены 8 публичных
   `float`-полей и 8 строк в секцию «Reward (this step)». Высота
   бокса увеличена с 380 до 540.

### Чего НЕ делал

- Python-сторону не трогал — layout наблюдений/действий не менялся.
- Логика геймплейных гейтов (когда можно прыгать, дашить и т.п.) не
  менялась — только добавлены счётчики **снаружи** этих гейтов.
- Никаких изменений в reward-весах для существующих компонентов
  (style/kills/parry/headshot/rank/damage/death/explore/survival).

## Калибровка коэффициентов (как это сейчас работает в цифрах)

| Компонент | Коэф. | Когда заметен |
|---|---|---|
| Jitter (camera) | -0.03 за единицу | На полной дрожи раз в кадр ≈ -0.06..-0.12/шаг |
| Jitter (move)   | -0.01 за единицу | Слабее, чтобы не наказывать стрейф |
| Wasted          | -0.02 за фронт   | -0.1..-0.2/шаг при «жму всё подряд» |
| Bank (dash)     | -0.05 за реальный даш | Один даш до килла = -0.05 vs +1 за килл |
| Bank (punch)    | -0.05 за реальный панч | То же |
| SlamAbuse       | -0.10 за событие | Сильный сигнал «не сам себя стопь» |
| SwitchSpam      | -0.05 за реальный свич | На 60-кадровом CD ≈ -0.05/секунду максимум |
| WhipSpam        | -0.03 за бросок  | Гарпун только когда реально нужен |
| FireTog         | -0.02 за лишний фронт | 1 фронт/шаг бесплатно |
| Mash            | -0.01 за лишнюю кнопку | Бесплатно до 3 одновременно |

Один килл (+1) перебивает ~20 «холостых» действий — то есть
осмысленные комбо в бою остаются положительными, а холостой спам уходит
в минус.

## Риски и что проверить вручную

### 1. Конфликт с устаревшим master

Пользователь сказал, что master на гитхабе сильно устарел. Я работал
с тем кодом, что был на ветке. Возможные проблемы:

- Если в локальной/новой версии `ActionExecutor.Execute` уже
  отрефакторен, конфликт будет в блоке дискретных действий
  (строки ~268-348). Решается по смыслу: сохрани новые гейты, оберни
  каждый удачный путь в `*Used++`, каждый отказ-по-фронту в
  `WastedActions++`.
- Если `TcpBridge.CalculateReward` уже изменён — нужно вручную
  встроить блок «Anti-spam penalties» и добавить компоненты в сумму
  `reward = ...` и в HUD.
- Если `DebugHUD` уже расширен — просто добавить недостающие 8 полей
  и строк.

### 2. Сборка

```
cd UltrabotMod/Plugin && dotnet build
```

Должен пройти под .NET Framework 4.7.1. Я это сделать не мог. При
ошибках компиляции наиболее вероятные места:
- неверное имя поля игры (`gc.onGround`, `boostCharge`, `falling`,
  `sliding`, `currentPunch.ready`, `fistCooldown`, `currentSlotIndex`,
  `currentVariationIndex`) — если изменилось в новой версии игры,
  смотри через `dll-inspector/` в репо.
- `_player.Slamdown(1f)` — сигнатура могла поменяться.

### 3. In-game проверка

- Скопировать DLL в `BepInEx/plugins/UltrabotMod/` (или `install.bat`).
- Запустить ULTRAKILL, загрузить уровень, в `UltrabotMod/Python`
  запустить `python train.py` или `play.py`.
- Нажать **F7** — на HUD должны появиться 8 новых строк
  (Jitter / Wasted / Bank / SlamAbuse / SwitchSpm / WhipSpm /
  FireTog / Mash).
- На свежей политике сразу видно крупные минусы по
  `Jitter / SlamAbuse`. При выключенном боте (F8) — все нули.
- Проверить регрессию: прыжок на земле всё ещё прыгает, fire1 всё ещё
  стреляет, `boostCharge` тратится только при реальном даше.
- Через ~30 минут тренировки на одном уровне:
  - дрожь прицела должна пропасть,
  - бот должен реально двигаться по карте,
  - банки даша/кулака не должны быть постоянно пустыми.

### 4. Тюнинг

Если анти-спам перебивает положительный сигнал (`CumulativeReward`
уходит в стабильный минус даже после первых киллов) — снизить
коэффициенты `rewBank`, `rewSwitchSpam`, `rewSlamAbuse` в 2 раза.
Структура от этого не страдает.
