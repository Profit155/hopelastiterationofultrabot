## Фаза 2: Исправление 10+ багов геймплея

### Баг 1: Actions обходили InputActionState
**Проблема**: Dash вызывал `Dodge()`, slide — `StartSlide()`, jump — `Jump()`, punch — `PunchStart()` через рефлексию напрямую. Только Fire1/Fire2 шли через InputActionState.
**Фикс**: Все действия переведены на InputActionState (Jump, Dodge, Slide, Punch, ChangeFist). Кэшируются 7 полей из PlayerInput.

### Баг 2: NavMesh (3 итерации)
- v1: Hard block через NavMesh.SamplePosition — бот замер
- v2: Мягкий lerp — бот ходил кругами
- v3 (финальная): Physics.Raycast + Vector3.ProjectOnPlane для скольжения вдоль стен

### Баг 3: Death penalty каждый кадр
**Проблема**: `info.Dead` true несколько кадров, штраф -5..-19 давался десятки раз.
**Фикс**: Флаг `isFirstDeathFrame`.

### Баг 4: Спавн на лифте
**Проблема**: `_spawnPos` захватывалась пока игрок на лифте (высоко).
**Фикс**: `_spawnPosLocked` — ждет `player.gc.onGround`.

### Баг 5: NavMesh GPS целился только в чекпоинты
**Фикс**: Приоритет: живой враг > чекпоинт > FinalDoor.

### Баг 6: Нет наблюдения "вижу врага"
**Фикс**: 3 фичи: nearest_enemy_dot, nearest_enemy_dist, has_enemy_los. PlayerFeatures 41→44.

### Баг 7: Нет aim hint
**Фикс**: 4 float (yaw_delta, pitch_delta, has_target, in_frustum). OBS_SIZE = 241.

### Баг 8: Нет смены кулака
**Фикс**: action[19] = ChangeFist. ActionSize 19→20.

### Баг 9: Дисбаланс наград
**Фикс**: Kill 1→5, Parry 3→5, Headshot 0.5→2, Exploration 0.05→0.01, Step cost 0.001→0.002.

### Баг 10: Railcannon и Punch не работают
**Проблема**: Оружия проверяют `WasPerformedThisFrame` (PerformedFrame == Time.frameCount). SetActionInput ставил PerformedFrame КАЖДЫЙ кадр, не давая rising edge.
**Фикс**: Edge detection через `_prevActionStates` — PerformedFrame только на rising edge, CanceledFrame на falling edge.

## Фаза 3: NavMeshAgent (добавлен и убран)

Попытка создать NavMeshAgent (как у врагов) для навигации.
- NavMeshAgent.speed=0 давал нулевой desiredVelocity
- "Failed to create agent because there is no valid NavMesh" спамило тысячи раз
- **Причина**: NavMesh в ULTRAKILL запечен для врагов, не для игрока
- **Важно**: Пользователь настоял не убирать навмеш — враги по нему нормально ходят
- **Результат**: Убран NavMeshAgent, оставлен NavMesh.CalculatePath в наблюдениях

## Фаза 7: rotationX/Y перепутаны (КОРНЕВАЯ ПРОБЛЕМА!)

### Открытие
В ULTRAKILL CameraController:
- `rotationX` = **PITCH** (вертикальная, вверх/вниз)
- `rotationY` = **YAW** (горизонтальная, влево/вправо)

Это было **перепутано ВЕЗДЕ** — в ActionExecutor, GameStateReader и TestPanel. Aim assist для yaw применялся к pitch → бот смотрел в пол/потолок вместо поворота к врагам.

### Фикс
Swap rotationX↔rotationY в:
- `ActionExecutor.cs` — 5 правок (yawDelta, pitchDelta, nextYaw, nextPitch, финальная запись)
- `GameStateReader.cs` — 4 правки (camYawRad, camPitchRad, yawDelta, pitchDelta)
- `TestPanel.cs` — 4 правки (Yaw Left/Right, Pitch Up/Down)

General-purpose агент выполнил swap в ActionExecutor и GameStateReader.

## Фаза 8: InputActionState из OnGUI (TestPanel)

### Проблема
`PressAction()` вызывался из `OnGUI()` (после Update). Игра уже обработала ввод на этом кадре → `WasPerformedThisFrame` = false когда игра проверяла.

### Фикс
- Кнопка ставит действие в `_queuedPresses` (HashSet)
- `Update()` (вызывается из MainLoop coroutine) применяет InputActionState в правильной фазе
- Re-press каждый кадр пока тест активен (чтобы WasPerformedThisFrame работал)
- Direct methods тоже через очередь `_queuedDirectCalls`

### Дополнительные фиксы TestPanel (по ревью Codex):
- Перекрывающиеся кнопки (Jump() поверх Blue Fist, StopMove поверх Red Fist)
- GUIStyle аллокации каждый кадр → кешированы в InitStyles()
- Сброс ссылок при смене сцены (проверка `_player == null`)
